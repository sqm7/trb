# Data List Report (交易資料明細)

## 1. 概述
`DataListReport` 元件負責展示詳細的房地產交易紀錄。它是數據分析的基礎視圖，允許用戶檢視原始數據，並支持多維度的篩選與排序。

## 2. 功能特點
- **列表展示**: 以表格形式展示每一筆交易的詳細資訊（建案名稱、樓層、單價、總價等）。
- **篩選功能**: 支援依據行政區、建案名稱、戶型、主要用途等條件進行關鍵字搜尋。
- **排序功能**: 點擊表頭可對關鍵欄位（日期、單價、總價等）進行升/降序排列。
- **分頁控制**: 支援大數據分頁瀏覽，預設每頁顯示 20 筆。
- **車位/附表彈窗 (`SubTable`)**: 針對每一筆交易，可點擊「附表」查看關聯的車位明細或建物分項明細。
- **明細展開**: 支援點擊「明細」在行內展開更多次要欄位資訊。
- **數據匯出**: 支援將當前篩選後的交易列表匯出為 CSV。

## 3. props 定義
- `data`: 初始數據（選填，相容性保留）。
- `trigger`: 觸發數據重新抓取的信號（通常來自上層 `AnalysisData` 更新）。

## 4. 關鍵邏輯
- **數據獲取 (`fetchByType`)**: 根據全域篩選器 (`useFilterStore`) 的狀態（縣市、行政區、日期、交易類型等），並行向後端請求各縣市數據。
- **子表查詢 (`handleFetchSubTable`)**: 單獨請求特定交易編號 (`id`) 的關聯車位與建物明細。
- **排序與篩選 (`processedData`)**: 在前端進行二次篩選與排序，以提供即時的交互體驗。
