---

### 執行步驟：

1. **掃描代辦清單**：
   - 執行 SQL 查詢全台 `{code}_projects` 表中，`enrichment_status = 'requested'` 的資料。
   - 優先處理 `last_enriched_at` 為空或時間最久遠的項目。

2. **啟動搜尋引擎**：
   - 針對每個案名，優先搜索「樂居」、「591」、「海悅」及「內政部建照執行系統」。
   - 提取欄位：投資建設、關係企業、營造廠、建築師、基地面積、公設比、總樓層、地下層、總戶數、結構、使用分區、建照號碼、代銷、車位類型、車位數量。

3. **數據驗證與清洗**：
   - 使用 AI 邏輯將非正規格式（如坪數、樓層數）轉換為標準數值。
   - 確保案名映射符合 `project_name_mappings` 規則。

4. **更新資料庫與狀態判定**：
   - 將獲獲取資料 Upsert 回對應的縣市表。
   - **狀態判定邏輯**：
     - 若 **除了「代銷公司」以外的 15 個欄位皆已補齊** ➡ 更新為 `'done'` (完備)。
     - 若有 **任何一個核心欄位缺失** (代銷除外) ➡ 更新為 `'pending'` (待補)，標記為 Agent 已查過但需要人工補回。
   - 更新 `last_enriched_at` 為目前時間。

5. **回報進度**：
   - 每一批次（預設 10 筆）完成後，向用戶回報成功筆數與變更摘要。
